Требования маркетолога неполные, и я бы попытался с ним связаться для уточнения деталей (если такая возможность есть).
В рамках задачи связи с маркетологом нет, поэтому сделаю допущение: у компании уже есть список городов с соответствующими номерами телефонов в БД.

Цель: определить город пользователя и отобразить корректный ему номер телефона.

**Вариант 1: На сайте реализован выбор города для пользователей (или его можно добавить)**

Это наиболее надежный вариант, т.к. пользователь сам сообщает свой город
- Для авторизованных храним выбор в БД
- Для неавторизованных - в сессии
- При первом посещении можем определить город по IP (PHP GeoIP или аналоги) и предложить его пользователю
- Если город не определен или для города нет номера в системе, показываем номер по умолчанию.

**Вариант 2: Выбора города нет и добавить его нельзя**

Остается только GeoIP (или аналоги)
- Определяем город по IP адресу
- Нормализуем данные в соответствии с нашей схемой (Например Moscow -> msk)
- Ищем такой город в нашей системе. Если есть - подставляем его, если нет - подставляем номер по умолчанию (например, номер главного офиса).

Если текущий номер телефона жестко прописан в HTML, то нужно его вынести в переменную и подставлять из бэкенда. В остальном правок на фронте не потребуется, т.к. вся логика на бэкенде.
